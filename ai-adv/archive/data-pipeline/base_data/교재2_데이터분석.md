📘 [학습 노트] 교재 2. 데이터 분석 (Numpy, Pandas, 텍스트 분석)
1. Numpy와 행렬 연산
AI 및 머신러닝 분야에서 필수적인 고성능 수치 연산 라이브러리입니다.
• 특징: C언어로 구현되어 파이썬 기본 리스트보다 벡터, 행렬, 텐서 연산 속도가 매우 빠릅니다.
• 배열 생성 (Array Creation):
    ◦ np.array([리스트]): 리스트를 배열로 변환.
    ◦ np.zeros((m,n)), np.ones((m,n)): 0 또는 1로 채워진 m×n 행렬 생성.
    ◦ np.arange(start, stop, step): 특정 범위의 수열 생성.
    ◦ np.random.randn(m,n): 표준정규분포를 따르는 난수 배열 생성.
• 주요 연산:
    ◦ 요소별 연산 (Elementwise): +, -, *, / 등 같은 위치의 원소끼리 연산합니다.
    ◦ 내적 (Dot Product): np.dot(v1, v2)를 사용하며, 벡터의 방향 유사도 측정에 활용됩니다.
    ◦ 노름 (Norm): np.linalg.norm(v)으로 벡터의 크기(거리)를 계산합니다.
    ◦ 행렬 곱: np.matmul(A, B) 또는 @ 연산자를 사용합니다.
    ◦ 전치 행렬 (Transpose): A.T로 행과 열을 바꿉니다.
2. Pandas 데이터 처리
표(Table) 형태의 정형 데이터를 다루는 핵심 라이브러리입니다.
• 데이터 구조:
    ◦ Series: 1차원 데이터 배열.
    ◦ DataFrame: 여러 Series가 결합된 2차원 테이블 구조.
• 기본 조작 및 I/O:
    ◦ 확인: df.shape(크기), df.columns(컬럼명), df.info()(요약), df.describe()(통계).
    ◦ 입출력: pd.read_csv(), pd.read_json()으로 불러오고 df.to_csv() 등으로 저장합니다.
• 데이터 필터링 및 변환:
    ◦ 조건 필터링: df[(조건1) & (조건2)] 형태로 원하는 데이터를 추출합니다 (AND: &, OR: |).
    ◦ 파생 변수: apply(lambda ...)를 사용해 새로운 컬럼을 만들거나, .str 접근자로 문자열을 처리합니다.
• 고급 데이터 처리:
    ◦ 그룹화: df.groupby('기준컬럼').agg(...)를 통해 그룹별 통계(평균, 합계 등)를 계산합니다.
    ◦ 병합 (Merge): df1.merge(df2, on='key', how='left')와 같이 SQL의 JOIN과 유사한 기능을 수행합니다.
    ◦ 피벗 테이블: pivot_table()을 사용해 엑셀처럼 행/열 기준으로 데이터를 요약합니다.
3. 텍스트 전처리 핵심 기법 (5단계 파이프라인)
분석 모델에 데이터를 넣기 전, 노이즈를 제거하고 표준화하는 필수 과정입니다.
1. 정규표현식 (Regex): \d+(숫자), [가-힣]+(한글) 등의 패턴을 사용해 원하는 텍스트를 검색하거나 추출합니다.
2. 정제 (Cleaning): HTML 태그, URL, 이모지, 특수문자 등 불필요한 노이즈를 제거합니다 (예: re.sub 활용).
3. 토큰화 (Tokenization): 문장이나 문서를 의미 있는 단위(단어, 형태소 등)로 분리합니다. 문맥 반영을 위해 N-gram(연속된 n개의 단어 묶음)을 사용하기도 합니다.
4. 정규화 (Normalization): 대소문자 통일, 숫자 치환(NUM), 반복 문자 축소 등을 통해 단어의 형태를 통일합니다.
5. 불용어 (Stopwords) 제거: "이", "그", "그리고" 등 분석에 큰 의미가 없는 조사나 접속사를 제거합니다.
4. 텍스트 데이터 변환과 통계 분석
컴퓨터가 이해할 수 있도록 텍스트를 수치(벡터)로 변환하는 과정입니다.
• Bag of Words (BOW):
    ◦ 단어의 순서를 무시하고 출현 빈도만 세어 벡터로 표현합니다.
    ◦ 구조가 단순하고 빠르지만, 문맥 정보가 손실되고 벡터가 희소(Sparse)해지는 단점이 있습니다.
• TF-IDF:
    ◦ 단순 빈도(TF)에 역문서 빈도(IDF)를 곱해 가중치를 줍니다.
    ◦ 특정 문서에서는 자주 나오지만 전체 문서에서는 희귀한 단어(중요 단어)를 강조할 때 사용합니다.
• 문서 유사도 측정:
    ◦ 코사인 유사도 (Cosine Similarity): 두 벡터의 방향(각도) 유사성을 측정하며, -1~1 사이의 값을 가집니다 (검색, RAG에서 주로 사용).
    ◦ 유클리드 거리: 두 벡터 간의 직선 거리를 측정합니다.
