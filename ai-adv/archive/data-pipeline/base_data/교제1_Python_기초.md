📘 [학습 노트] 교재 1. Python 기초
1. 파이썬 개요 및 개발 환경
• 파이썬의 특징: 문법이 간결하고 배우기 쉬우며, 웹 개발, 데이터 분석, AI/ML 등 범용적으로 사용됩니다. 특히 방대한 라이브러리와 커뮤니티가 강점입니다,.
• 설치 및 실행:
    ◦ 설치 시 "Add python.exe to PATH" 옵션을 반드시 체크해야 터미널에서 실행이 원활합니다.
    ◦ 실행 방법: 터미널(python --version), VS Code 등에서 실행 가능합니다.
• 파일 형식:
    ◦ .py: 일반적인 파이썬 스크립트 파일로 실제 개발에 적합합니다.
    ◦ .ipynb: 주피터 노트북 형식으로 코드, 실행 결과, 문서를 함께 관리할 수 있어 데이터 분석 및 교육에 적합합니다.
• 가상환경: 프로젝트별로 라이브러리 버전 충돌을 방지하기 위해 독립적인 가상환경을 생성하여 사용합니다.
2. 변수와 자료형
• 변수(Variable):
    ◦ 데이터를 저장하는 메모리 공간으로, 파이썬은 실행 시점에 타입이 결정되는 동적 타이핑(Dynamic Typing) 언어입니다,.
    ◦ PEP8 명명 규칙: 의미 있는 이름을 사용하며, 변수나 함수명은 소문자와 언더바(_)를 사용하고, 상수는 대문자(MAX_RETRY)로 표기합니다.
• 기본 자료형:
    ◦ int/float: 정수(메모리 제한 없는 큰 수 가능)와 실수(부동소수점).
    ◦ str: 문자열, 불변(immutable) 시퀀스입니다.
    ◦ bool: True/False (내부적으로 1/0).
    ◦ None: 값이 없음을 표현하며 초기값이나 실패 표시에 사용됩니다.
• 컬렉션 자료형:
    ◦ list: [] 사용, 순서가 있고 값 변경이 가능(mutable)합니다.
    ◦ tuple: () 사용, 순서가 있으나 값 변경이 불가능(immutable)합니다.
    ◦ dict: {키: 값} 구조, 키를 통해 값을 빠르게 조회합니다.
• 형변환 및 타입 확인: int(), str() 등으로 형변환하며, type(), isinstance()로 타입을 확인합니다.
3. 제어문 (조건문과 반복문)
• 조건문 (if): if, elif, else 구조를 가집니다.
    ◦ Truthy/Falsy: 파이썬에서는 None, False, 0, ""(빈 문자열), [](빈 리스트) 등은 거짓(Falsy)으로, 그 외는 참(Truthy)으로 취급됩니다,.
    ◦ 논리 연산: and, or, not을 사용합니다.
• 반복문:
    ◦ for: range(), enumerate()(인덱스+값), zip()(여러 시퀀스 동시 순회)과 함께 자주 사용됩니다.
    ◦ while: 조건이 True인 동안 반복하며, 무한 루프에 빠지지 않도록 조건 변경 로직이 필수입니다.
    ◦ 제어 키워드: break(종료), continue(다음 반복으로 건너뜀), pass(아무 동작 안 함).
4. 함수와 모듈
• 함수 (Function):
    ◦ DRY 원칙: 중복 코드를 제거(Don't Repeat Yourself)하고 재사용성을 높이기 위해 사용합니다,.
    ◦ 매개변수: 위치 인자, 키워드 인자(name="철수"), 기본값 매개변수, 가변 인자(*args-튜플, **kwargs-딕셔너리) 등을 지원합니다.
• 모듈과 패키지:
    ◦ 모듈: .py 파일 하나가 모듈이며 import로 재사용합니다.
    ◦ 패키지: 여러 모듈을 모아둔 폴더(__init__.py 포함)입니다.
    ◦ 라이브러리: 모듈과 패키지의 묶음으로 pip install을 통해 외부 라이브러리(numpy, pandas 등)를 설치해 사용합니다.
5. 자료구조 심화 및 파일/예외 처리
• 리스트 컴프리헨션: 반복문과 조건을 한 줄로 간결하게 리스트를 생성하는 문법입니다.
    ◦ 예: [x**2 for x in range(10) if x % 2 == 0].
• 파일 입출력:
    ◦ open() 함수와 with 문을 함께 사용하여 파일을 열고 자동으로 닫히게 처리하는 것이 권장됩니다.
    ◦ 모드: r(읽기), w(쓰기), a(추가).
• JSON 처리: json.dumps()(딕셔너리→문자열), json.loads()(문자열→딕셔너리)를 통해 데이터를 교환합니다.
• 예외 처리:
    ◦ 프로그램 비정상 종료를 막기 위해 try ~ except 구문을 사용합니다.
    ◦ 구체적인 예외부터 처리하고, finally 블록은 예외 여부와 상관없이 항상 실행됩니다.
6. 객체지향 프로그래밍 (OOP) 기초
• 클래스와 인스턴스:
    ◦ Class: 데이터(속성)와 기능(메서드)을 묶은 설계도입니다.
    ◦ Instance: 클래스를 통해 생성된 실체입니다.
    ◦ 생성자(__init__): 인스턴스 생성 시 초기화를 담당하며, self는 인스턴스 자신을 가리킵니다.
• 변수의 종류:
    ◦ 클래스 변수: 모든 인스턴스가 공유하는 변수입니다.
    ◦ 인스턴스 변수: self.변수명으로 선언하며 각 객체별로 독립적인 값을 가집니다.
• 상속과 오버라이딩:
    ◦ 상속: 부모 클래스의 기능을 물려받아 재사용하거나 확장합니다.
    ◦ 오버라이딩: 자식 클래스에서 부모의 메서드를 재정의하는 것입니다.
    ◦ super(): 부모 클래스의 메서드를 호출하여 기능을 유지하면서 추가 코드를 작성할 때 사용합니다.
