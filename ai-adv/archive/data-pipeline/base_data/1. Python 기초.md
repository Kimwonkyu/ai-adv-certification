## 1. 파이썬 개요·환경 

- 파이썬 특징: 문법 간결·배우기 쉬움 / 방대한 라이브러리·커뮤니티 / 범용 언어(웹, 데이터분석, AI/ML, 자동화 등).
- 설치 포인트: python.org에서 설치, "Add python.exe to PATH" 꼭 체크.
- 실행 방법: 터미널에서 `python`, `python --version`, VS Code에서 .py 파일 실행.
- 가상환경: 프로젝트별 라이브러리 충돌 방지를 위해 가상환경 생성하여 사용.
- .py vs .ipynb
  - .py: 일반 스크립트, 개발에 적합.
  - .ipynb: 노트북 형식, 코드+결과+문서, 분석 교육 실험에 적합.

## 2. 변수·자료형·제어문

### 변수·명명 규칙

  - 변수: 데이터를 저장하는 메모리 공간, 파이썬은 동적 타이핑(실행 시 타입 결정).
  - 키워드(var, let, const) 없음, 타입 자유롭게 변경 가능.
  - PEP8 규칙: 의미 있는 이름, 예약어(def, if 등) 금지, 특수문자는 언더바(_)만, 상수는 대문자 사용(MAX_RETRY).

### 기본 자료형
  - int: 정수, 메모리 제한 없는 큰 수, 카운터·인덱스·ID 등에 사용.
  - float: 실수, 부동소수점(정밀도 한계), 비율·측정값·과학 계산에 사용.
  - str: 문자열, 불변 시퀀스, 텍스트·로그에 사용.
  - bool: True/False(내부 1/0), 플래그·조건 판단.
  - None: 값 없음 표현, 초기값·실패 표시.

### 컬렉션 자료형

  - list: `[]`, 순서 있음, 값 변경 가능, 다양한 타입 섞어서 저장 가능.
  - tuple: `()`, 순서 있음, 값 변경 불가.
  - dict: `{키:값}`, 키로 값 조회, 예) `{'name':'서두석','age':27}`.

### 형 확인·형변환

  - 타입 확인: `type(x)`, `isinstance(x, 타입)`. 
  - 형변환: `int("123")`, `str(100)` 등.

### 조건문·논리연산·Truthy/Falsy

  - if 구조: `if / elif / else`로 분기.
  - 논리연산자: `and`(모두 참), `or`(하나라도 참), `not`(반전).
  - Falsy: `None, False, 0.0, ""(빈 문자열), [], (), {}`.
  - Truthy: 위 Falsy를 제외한 모든 값.

### 반복문(for, while)

  - for: `for 변수 in 이터러블:`; range, enumerate, zip 자주 사용.
  - range: 숫자 시퀀스 생성 / enumerate: 인덱스+값 / zip: 여러 시퀀스 동시 순회.
  - while: 조건이 True인 동안 반복, 조건 변경 로직 필수.
  - 제어 키워드: `breake`(즉시 종료), `continue`(다음 반복), `pass`(아무 것도 안 함), `else`(break 없이 정상 종료 시 실행).

## 3. 함수·모듈·패키지

### 함수 기본

  - 정의: 입력 → 처리 → 출력(반환값)을 갖는 재사용 가능한 코드 블록.
  - 설계 원칙: DRY(중복 코드 제거), 재사용성과 가독성 중시.
  - 구성: 시그니처(def 함수명(매개변수)), docstring, 본문(로직), `return`.

### 매개변수 종류

  - 위치 인지: 순서대로 값 전달.
  - 키워드 인자: `greet(age=25, name="김철수")`처럼 이름으로 전달, 순서 무관.
  - 기본값 매개변수: `def connect(host="localhost", port=3306)`처럼 기본값 지정 가능.
  - 가변 매개변수: `*args`(튜플), `**kwargs`(딕셔너리).

### 모듈·패키지·라이브러리

  - 모듈: `.py` 파일 하나가 모듈, `import`로 다른 곳에서 사용.
  - 패키지: 폴더 단위 모음(`__init__.py` 포함), 내부에 여러 모듈.
  - 라이브러리: 모듈·패키지 묶음, 예) numpy, pandas, transformers 등, `pip install`로 설치.

## 4. 자료구조 심화·파일·예외

### 리스트 컴프리헨션

- 목적: 반복문+조건을 한 줄로 간결하게 표현.
- 예: `result = [x**2 for x in range(10) if x % 2 == 0]`.
  
### 파일 입출력

- `open()` 사용, 모드: `"r"(읽기)`, `"w"(쓰기)`, `"a"(추가)`, `"b"(바이너리)`.
- with 패턴: `with open("example.txt","r", encoding="utf-8) as f: `로 자동 close.

### JSON 처리

- JSON: 경량 데이터 교환 형식, 파이썬 dict와 구조 유사.
- `json.dumps()`로 dict→문자열, `json.loads()`로 문자열→dict.

### 예외 처리

- 철학: if로 전부 방어하기보다는 실제 발생한 예외를 try-except로 처리.
- 구조: `try / except 구체예외 / except Exception as e / else / finally`.

## 5. 객체지향 기초

### 클래스·인스턴스

- Class: OOP의 기본 단위, 속성(데이터)+메서드(기능)를 묶은 것.
- 생성자 `__init__`: 인스턴스 생성 시 초기화, `self`로 인스턴스 자신을 가리킴.
- 예: `Person("철수")` 생성 후 `p.greet()` 호출.

### 클래스 변수 vs 인스턴스 변수

- 클래스 변수: 모든 인스턴스가 공유, `클래스명.변수`로 접근.
- 인스턴스 변수: 각 객체별 독립, `self.변수`로 정의.

### 상속·오버라이딩·super()

- 상속: 부모 클래스 기능 재사용 + 자식에서 기능 확장.
- 오버라이딩: 자식이 부모 메서드를 재정의.
- `super()`: 부모 네서드 호출 후 기능 덧붙일 때 사용.
