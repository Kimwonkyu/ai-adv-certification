# 📘 [학습 노트] 교재 1. Python 기초

## 1. 파이썬 개요 및 개발 환경 (Deep Dive)
파이썬은 **"인터프리터 프로토타입 언어"**에서 시작하여 현재는 AI와 데이터 과학의 표준이 된 범용 언어입니다.

### 🐍 파이썬의 핵심 철학 (Zen of Python)
- "Beautiful is better than ugly." (아름다운 것이 추한 것보다 낫다)
- "Simple is better than complex." (단순한 것이 복잡한 것보다 낫다)
- **가독성(Readability)**이 생산성으로 직결된다는 믿음을 바탕으로 설계되었습니다.

### 🛠 현대적인 개발 환경 설정
1.  **가상환경 (Virtual Environments)**: 프로젝트별로 독립적인 라이브러리 환경을 구축하여 버전 충돌을 방지합니다.
    - `venv`: 파이썬 표준 라이브러리 (`python -m venv venv`)
    - `Conda`: 데이터 사이언스 필수 패키지와 하드웨어 종속 라이브러리(CUDA 등) 관리에 용이합니다.
2.  **포맷팅 및 린팅 (PEP8)**:
    - 파이썬 공인 스타일 가이드(PEP8)를 준수하는 것이 협업의 기본입니다.
    - `Black`(포맷터), `Flake8`(린터) 등을 VS Code에 설정하여 자동 교정하는 것이 권장됩니다.

## 2. 변수와 자료형: 데이터의 저장과 흐름
파이썬은 **동적 타이핑(Dynamic Typing)** 언어이지만, 최근에는 `Typing` 모듈을 통한 **Type Hint** 사용이 권장됩니다.

### 📦 자료형의 분류 (Mutable vs Immutable)
이 구분은 함수 파라미터 전달 시 데이터 원본의 훼손 여부를 결정하므로 매우 중요합니다.
- **불변(Immutable)**: `int`, `float`, `str`, `tuple`. 생성 후 값을 바꿀 수 없습니다.
- **가변(Mutable)**: `list`, `dict`, `set`. 생성 후에도 내부 값을 자유롭게 변경 가능합니다.

### 💡 실무 Tip: 복사(Copy)의 함정
- **얕은 복사(Shallow Copy)**: 리스트 내부의 중첩된 객체는 주소만 복사되어 원본이 바뀔 수 있습니다.
- **깊은 복사(Deep Copy)**: `copy.deepcopy()`를 사용하여 완전한 독립 객체를 생성해야 안전합니다.

## 3. 강력한 제어문과 컴프리헨션
단순 반복을 넘어 선언적인 코드 작성을 지향합니다.

### 🚀 리스트 컴프리헨션 (List Comprehension)
반복문과 조건을 연결하여 간결하게 리스트를 생성합니다.
```python
# 일반적인 방식
squares = []
for x in range(10):
    if x % 2 == 0:
        squares.append(x**2)

# 컴프리헨션 방식 (권장)
squares = [x**2 for x in range(10) if x % 2 == 0]
```

### 🔍 Truthy & Falsy
- 거짓(`False`)으로 판정되는 값: `None`, `0`, `0.0`, `""`, `[]`, `{}`, `set()`
- 이를 활용하여 `if my_list:` 처럼 명시적으로 비어있는지 체크하는 코드가 파이썬스럽습니다(Pythonic).

## 4. 함수와 객체지향 (OOP)
코드의 재사용성과 확장성을 확보하는 핵심입니다.

### 🛠 함수 (Function)의 심화 기능
- **가변 인자**: `*args`(튜플로 수신), `**kwargs`(딕셔너리로 수신).
- **데코레이터 (Decorator)**: 함수를 수정하지 않고 기능을 추가하는 강력한 기법입니다 (예: 로그 기록, 권한 확인).

### 🏛 클래스 (Class) 설계 원칙
- **상속 (Inheritance)**: 부모 클래스의 기능을 확장.
- **오버라이딩 (Overriding)**: 부모의 메서드를 자식 식에 맞게 재정의.
- **super()**: 부모 인스턴스의 메서드를 안전하게 호출.
```python
class Model:
    def __init__(self, name):
        self.name = name
    def run(self):
        print(f"{self.name} 실행 중")

class GPT(Model):
    def run(self):
        super().run() # 부모 로직 유지
        print("생성 AI 엔진 활성화")
```

## 5. 예외 처리 및 비동기 기초
프로그램의 안정성을 높이는 마지막 단계입니다.

### 🛡 예외 처리 (Try-Except-Finally)
- 구체적인 에러(`ValueError`, `KeyError`)를 먼저 잡고, `finally`를 통해 자원(파일, DB 연결)을 해제합니다.
- `with` 문(Context Manager)을 사용하면 `open()` 등의 자원 해제를 자동으로 처리해 줍니다.

### 🌐 비동기 프로그래밍 (Async/Await)
- I/O 작업(웹 요청, DB 쿼리) 대기 시간 동안 다른 작업을 수행하여 효율을 높입니다.
- AI 라이브러리(LangChain, FastAPI)에서 멀티 에이전트 구동 시 필수적으로 사용됩니다.
