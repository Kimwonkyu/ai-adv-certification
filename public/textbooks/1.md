## 1. 파이썬 입문 및 환경 설정
파이썬은 1991년 귀도 반 로섬이 발표한 인터프리터 방식의 고급 프로그래밍 언어입니다.

### 🐍 파이썬의 특징
- **간결한 문법**: 가독성을 중시하며, 들여쓰기(Indentation)가 문법의 필수 요소입니다.
- **동적 타이핑(Dynamic Typing)**: 변수 선언 시 타입을 명시하지 않고 실행 시점에 결정됩니다.
- **방대한 라이브러리**: 'Batteries Included' 철학에 따라 기본 라이브러리가 풍부하며, PyPI를 통해 40만 개 이상의 외부 패키지를 사용할 수 있습니다.

### 🛠 환경 설정 핵심
- **PEP 8**: 파이썬의 공식 스타일 가이드입니다. 
    - 들여쓰기는 공백 4칸을 권장합니다.
    - 변수/함수명은 `snake_case`, 클래스명은 `PascalCase`를 사용합니다.
- **가상환경**: 프로젝트별 의존성 격리를 위해 사용합니다.
    - `venv`: 표준 모듈 (`python -m venv .venv`)
    - `pip`: 패키지 관리자 (`pip install -r requirements.txt`)
- **실행 모드**:
    - `.py`: 스크립트 모드
    - `.ipynb`: 대화형(Interactive) 모드 (Jupyter Notebook)
    - `if __name__ == "__main__":`: 모듈이 직접 실행될 때만 작동하는 코드 블록입니다.

## 2. 자료형과 변수의 이해 (Deep Dive)
파이썬의 모든 것은 **객체(Object)**입니다.

### 📦 데이터 타입 분류
1.  **Numeric**: `int`(정수 - 크기 제한 없음), `float`(실수 - 부동소수점 방식)
2.  **Sequence**: `str`(불변), `list`(가변), `tuple`(불변), `range`
3.  **Mapping**: `dict`(가변, Key-Value)
4.  **Set**: `set`(가변, 중복 불가, 순서 없음)

### 🚨 Mutable vs Immutable (가변 vs 불변)
이 개념은 파이썬 메모리 관리의 핵심입니다.
- **Immutable(불변)**: `int`, `float`, `str`, `tuple`, `bool`. 값을 수정하면 새로운 객체가 생성됩니다.
- **Mutable(가변)**: `list`, `dict`, `set`. 기존 객체의 내용을 직접 수정할 수 있습니다.

```python
# Immutable 예시
s = "hello"
print(id(s))
s += " world"
print(id(s)) # ID가 바뀜 (새 객체 생성)

# Mutable 예시
l = [1, 2]
print(id(l))
l.append(3)
print(id(l)) # ID가 같음 (원본 수정)
```

### 💡 문자열(str) 주요 메서드
- `strip()`: 양끝 공백 제거
- `split(sep)`: 구분자로 나누어 리스트 반환
- `join(list)`: 리스트 원소들을 문자열로 합침
- `format()` / `f-string`: 문자열 포맷팅
- `find()` vs `index()`: 찾기에 실패했을 때 `find`는 -1을 반환하고, `index`는 에러를 발생시킵니다.

## 3. 제어문과 반복문
코드의 흐름을 제어하는 핵심 구문입니다.

### 🔍 조건문 (if)
- `in`, `is`, `and`, `or`, `not` 연산자를 활용합니다.
- **Walrus Operator (`:=`)**: 파이썬 3.8+ 도입. 조건문 내에서 변수를 할당하고 동시에 검사할 수 있습니다.
```python
if (n := len(data)) > 10:
    print(f"너무 긴 데이터: {n}")
```

### 🔄 반복문 (for, while)
- **enumerate()**: 인덱스와 값을 동시에 반환합니다.
- **zip()**: 여러 시퀀스를 병렬로 순회합니다.
- **break / continue / pass**:
    - `break`: 루프 즉시 탈출
    - `continue`: 다음 반복으로 건너뜀
    - `pass`: 구체적인 구현을 나중에 하기 위한 빈 칸 (에러 방지)

## 4. 함수와 스코프 (Function & Scope)
### 🛠 파라미터 전달 방식
- **위치 인자** vs **키워드 인자**
- **기본값 파라미터**: 기본값으로 리스트와 같은 가변 객체를 주면 안 됩니다. (정적 메모리 공유 이슈)
- **가변 인자**:
    - `*args`: 여러 위치 인자를 튜플로 받음
    - `**kwargs`: 여러 키워드 인자를 딕셔너리로 받음

### 🌐 변수 범위 (Scope)
- **LEGB 규칙**: Local → Enclosing → Global → Built-in 순서로 검색합니다.
- `global` 키워드: 함수 내에서 전역 변수를 수정할 때 사용.
- `nonlocal` 키워드: 중첩 함수에서 상위 함수의 변수를 수정할 때 사용.

## 5. 객체지향 프로그래밍 (OOP)
### 🏛 클래스 핵심 요소
- **`self`**: 인스턴스 자신을 가리키는 첫 번째 매개변수입니다.
- **생성자 (`__init__`)**: 객체 생성 시 초기화를 담당하는 매직 메서드입니다.
- **클래스 변수** vs **인스턴스 변수**: 클래스 변수는 모든 인스턴스가 공유하며, 인스턴스 변수는 `self`를 통해 독립적으로 관리됩니다.
- **상속**: `class Child(Parent):` 구조로 기능을 물려받습니다. `super()`를 통해 부모 메서드에 접근합니다.
- **캡슐화**: 변수명 앞에 `__`(더블 언더바)를 붙여 private하게 관리할 수 있습니다 (이름 장식-Name Mangling 발생).

### ✨ 매직 메서드 (Dunder Methods)
- `__str__`: `print()` 시 출력될 문자열 정의
- `__len__`: `len()` 호출 시 반환값 정의
- `__getitem__`: 인덱싱(`[]`) 동작 정의

## 6. 예외 처리와 파일 입출력
### 🛡 예외 처리 (Exception)
```python
try:
    # 실행할 코드
except ValueError as e:
    # 특정 에러 처리
else:
    # 에러가 발생하지 않았을 때 실행
finally:
    # 에러 발생 여부와 상관없이 무조건 실행 (자원 해제)
```

### 📂 파일 및 JSON
- **`with open(...) as f:`**: 파일을 열고 작업이 끝나면 자동으로 닫아주는 Context Manager입니다.
- **JSON 처리**:
    - `json.dumps()`: 객체 → 문자열 (직렬화)
    - `json.loads()`: 문자열 → 객체 (역직렬화)

## 7. 심화 개념 (Advanced)
### 🚀 리스트 컴프리헨션
- `[f(x) for x in list if condition]` 구조로 성능과 가독성을 모두 잡습니다.

### 🧊 제너레이터 (Generator)
- `yield`를 사용하여 데이터를 하나씩 생성합니다. 메모리를 효율적으로 사용하며 대용량 처리에 필수적입니다.

### 🎭 데코레이터 (Decorator)
- `@decorator` 문법을 사용하여 함수의 기능을 확장합니다.

### 🧬 복사 (Copy)
- **얕은 복사 (Shallow Copy)**: `l.copy()` 또는 `l[:]`. 중첩된 가변 객체는 주소를 공유합니다.
- **깊은 복사 (Deep Copy)**: `copy.deepcopy()`. 모든 하위 객체까지 새로 생성합니다.
