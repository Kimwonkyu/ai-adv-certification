# 📘 [학습 노트] 교재 4. 프롬프트 엔지니어링 (Foundation to Advanced)

이 교재는 결과물 제어를 위한 기초 프롬프트 구조부터 논리적 추론 유도, 에이전트 설계, 그리고 상용 수준의 보안 전략까지 완벽히 아우릅니다.

---

## 1. 프롬프트 엔지니어링의 정석
### 🏗 프롬프트의 4대 구성 요소
1.  **Instruction (지시)**: 모델이 수행할 구체적인 작업.
2.  **Context (맥락)**: 답변 품질을 높이기 위한 배경 지식.
3.  **Input Data (입력 데이터)**: 처리가 필요한 실제 텍스트.
4.  **Output Indicator (출력 형식)**: JSON, 표 등 원하는 포맷 명시.

### 🏷 구조화 및 기법 개요
- **구분자 (Delimiter)**: `###`나 XML 태그(`<context>`)를 사용하여 데이터와 지시를 명확히 분리.
- **Zero-shot vs Few-shot**: 예시 없이 바로 묻는 것과, 몇 개의 예시(Shot)를 주어 패턴을 학습시키는 차이.

---

## 2. 심화 추론 유도 전략 (Logic)
### 🧠 Reasoning Techniques
- **CoT (Chain-of-Thought)**: "단계별로 생각해보자" 문구로 추론 경로를 활성화.
- **Few-shot CoT**: 정답뿐만 아니라 '풀이 과정'을 예시로 제공하여 논리적 정답률 극대화.
- **Self-Consistency**: 여러 추론 경로 중 다수결로 최종 정답 선택.

---

## 3. 에이전트와 ReAct 프레임워크
- **ReAct**: `Thought (생각) -> Action (행동) -> Observation (관찰)` 루프를 통해 자율적으로 도구(API, 검색기)를 사용함.
- **Tool Calling (Function Calling)**: 모델이 실행할 함수 인자(JSON)를 생성하면, 시스템이 실제 실행 후 결과를 돌려주는 구조.

---

## 4. 프롬프트 보안 (Defense)
### ⚔️ 공격 패턴
- **Prompt Injection**: 사용자 입력이 시스템 지시를 무시하게 만듦.
- **System Prompt Leaking**: 내부 기밀 지시사항 유출 유도.

### 🛡 방어 전략
- **XML 태그 사용**, **Post-Instruction**(지시사항을 뒤에 배치), **Output Validation**(출력물 형식 검사).

---

## 5. 프롬프트 프로그래밍: DSPy 및 LCEL
- **DSPy**: 프롬프트를 수동 작성 대신 코드로 정의하고 최적화(Compile)하는 프레임워크.
- **LCEL (LangChain Expression Language)**: `|` (Pipe) 연산자로 복잡한 체인을 선언적으로 결합.

이상의 내용은 프롬프트 엔지니어링 단원의 모든 기초 및 심화 문항에 대한 완벽한 정답 가이드가 됩니다.
